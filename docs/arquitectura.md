# Decisiones Arquitectonicas 
 
## Clean Architecture 
 
El proyecto sigue Clean Architecture con separacion de capas: 
 
- **delivery/rest**: Controladores REST 
- **domain/service**: Logica de negocio 
- **domain/repository**: Acceso a datos 
- **domain/model**: Entidades JPA 
 
## Patrones de Diseżo 
 
- Repository Pattern 
- Dependency Injection 
- DTO Pattern (para futuras implementaciones)
